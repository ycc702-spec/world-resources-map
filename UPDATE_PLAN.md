# AI資源地圖更新計畫

## 自動更新機制設計

### 1. 資料來源與更新頻率

| 資料類型 | 來源 | 更新頻率 | 自動化程度 |
|---------|------|---------|-----------|
| **天然氣儲量/產量** | BP Statistical Review | 年度（6月） | 🔶 半自動 |
| **礦產儲量** | USGS Mineral Commodity Summaries | 年度（1月） | 🔶 半自動 |
| **能源價格** | EIA / Bloomberg | 即時/日 | 🟢 可API自動 |
| **AI公司能耗** | 各公司CSR報告 | 年度（Q1-Q2） | 🔴 需手動 |
| **AI資料中心位置** | 新聞/公司公告 | 即時 | 🔴 需手動 |
| **碳排放數據** | CDP / 公司報告 | 年度 | 🔴 需手動 |

### 2. 建議的更新流程

```
┌─────────────────────────────────────────────────────────────┐
│                    自動更新排程                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  🕐 每日檢查（自動）                                         │
│     ├── 能源價格API（Henry Hub, TTF, JKM）                   │
│     └── 匯率變動                                            │
│                                                             │
│  📅 每月檢查（半自動）                                       │
│     ├── 新AI資料中心開設新聞                                 │
│     └── 地緣政治風險變化                                     │
│                                                             │
│  📊 每季檢查（手動）                                         │
│     ├── AI公司財報能耗數據                                   │
│     └── 各國能源政策變動                                     │
│                                                             │
│  🗓️ 年度大更新（手動）                                       │
│     ├── BP Statistical Review（6月）                         │
│     ├── USGS Mineral Summaries（1月）                        │
│     └── 各公司CSR報告（Q1-Q2）                               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3. 技術實現方案

#### 方案A：簡易版（推薦）
- 使用 GitHub Actions 排程
- 每月自動檢查API並更新價格數據
- 年度手動更新儲量數據

#### 方案B：進階版
- 建立後端資料庫（Firebase/Supabase）
- 每日自動抓取API數據
- 前端即時顯示最新價格

### 4. 建議的更新責任分工

| 項目 | 建議責任人 | 時間投入 |
|------|-----------|---------|
| 能源價格更新 | 自動化腳本 | 0人天 |
| 年度儲量更新 | 研究員/分析師 | 2-3人天/年 |
| AI公司能耗 | 研究員/分析師 | 5-7人天/年 |
| 新資料中心 | 研究員/分析師 | 1-2人天/月 |
| 風險評估更新 | 地緣政治分析師 | 2-3人天/季 |

### 5. 資料品質檢核清單

- [ ] 數據來源是否明確標註
- [ ] 更新日期是否顯示
- [ ] 數據變動是否合理（年度變化<20%）
- [ ] 單位是否一致
- [ ] 全球占比總和是否合理（約100%）

### 6. 實施建議

**短期（1-3個月）**
1. 建立能源價格API自動更新
2. 設置GitHub Actions每月提醒
3. 建立更新日誌

**中期（3-6個月）**
1. 建立資料驗證機制
2. 設置異常警報
3. 建立版本控制

**長期（6-12個月）**
1. 考慮建立後端資料庫
2. 實現即時價格顯示
3. 建立用戶回饋機制

---

## 結論

建議採用 **簡易版方案**：
- 能源價格：自動更新（每月）
- 儲量/能耗：手動更新（每年）
- 新聞事件：手動更新（按需）

預估年度維護成本：10-15人天
